# 置換・正規化ルール整理（transliteration / translation）

最終更新: 2026-03-16

このリポジトリ内で散在している「置換・正規化（pre/post）」観点を、**入力（transliteration）**と**出力（translation）**に分けて一覧化する。

- 公式/host 推奨（@deeppast Entry `678899`）の一次メモ: `.codex/docs/discussion_comments.md`
- Dataset Instructions（公式の表記方針）: `.codex/docs/dataset_instructions.md`
- 実装例（推論ノート）: `notebooks/006/lb-35-9-ensembling-post-processing-baseline.ipynb`（`OptimizedPreprocessor` / `VectorizedPostprocessor`）

> 注意: “学術的に正しい” 置換が LB で必ず有利とは限らないため、**host 推奨の test 整合**と、**ノートブック独自のクリーニング**を分けて扱う（ablation 前提）。

---

## 1) Transliteration（入力側）— 置換観点

### A. host（@deeppast）推奨の test 整合（Entry `678899`, 2026-02-26）

- **Gap 統一（欠損表現 → `<gap>`）**
  - 例: `x`, `[x]`, `…`, `(break)`, `(large break)`, `(n broken lines)` → `<gap>`
  - 連続 `<gap>` は 1 個に縮約（`<gap> <gap>` → `<gap>`）
- **決定詞（determinatives）の表記を test 側に寄せる**
  - 例: `(d)` → `{d}`, `(ki)` → `{ki}`, `(TÚG)` → `TÚG`
  - 実装上は「大文字決定詞はカッコを外す」「小文字決定詞は `{}` にする」等のルールで近似されることがある
- **長すぎる小数の短縮**（小数点以下 4 桁）
  - 例: `1.3333300000000001` → `1.3333`
- （任意）**音価記号の単純化**
  - 例: `Ḫ` → `H`, `ḫ` → `h`
- （任意）**略記の展開**
  - 例: `KÙ.B.` → `KÙ.BABBAR`
- （任意）**Unicode 下付き数字の正規化**
  - 例: `₀..₉` → `0..9`
- （任意）**小数→Unicode分数**
  - 例: `0.3333` → `⅓`, `0.6666` → `⅔`, `0.5` → `½` など

### B. Dataset Instructions に由来する観点（公式データの“意図”）

- **決定詞は `{}` で保持される**（分類子として意味を持つ可能性）
  - 例: `a-lim{ki}`, `A-mur-{d}UTU`
- **角括弧 `[]` は document-level で除去されることがある**
  - 例: `[KÙ.BABBAR]` → `KÙ.BABBAR`

### C. 実装例（`lb-35-9` ノート）に見られる追加の入力正規化

`notebooks/006/lb-35-9-ensembling-post-processing-baseline.ipynb` の `OptimizedPreprocessor`（= この repo 内での **実装の一次情報**）は、概ね「test 整合」を満たしつつ、次の順で処理している。

#### C-1. ASCII風の表記→ダイアクリティクス（入力）

- `sz/SZ` → `š/Š`
- `s,/S,` → `ṣ/Ṣ`
- `t,/T,` → `ṭ/Ṭ`
- 母音 `a/e/i/u/A/E/I/U` の直後に `2/₂` が付く場合は **アキュート**に変換（例: `a2`→`á`）
- 母音 `a/e/i/u/A/E/I/U` の直後に `3/₃` が付く場合は **グレイヴ**に変換（例: `u3`→`ù`）

#### C-2. 決定詞（determinatives）

- **大文字決定詞**: `(X...)` → `X...`（括弧を外す）
- **小文字決定詞**: `(d)` / `(ki)` 等 → `{d}` / `{ki}`（波括弧へ）

#### C-3. Gap 統一（欠損表現 → `<gap>`）

以下をすべて `<gap>` に正規化（`lb-35-9` の `_GAP_UNIFIED_RE` 相当）:

- `<big gap>` / `<big_gap>` / `<big-gap>`（空白/`_`/`-` の揺れを吸収）
- `<gap>`
- `big gap`（空白/`_`/`-` の揺れを吸収）
- `x x x` のような “x の並び” / `xx` / 単独 `x`
- `...`（3点以上）, `…`, `[...]` 的な省略
- `[x]` / `(x)`
- `(break)` / `(large break)` / `(<n> broken line[s])`

#### C-4. 文字正規化（入力）

- 文字置換:
  - `ḫ/Ḫ` → `h/H`
  - `ʾ` → **削除**
  - 下付き数字 `₀..₉` → `0..9`
  - `—/–` → `-`
- 下付き `ₓ` を削除
- `KÙ.B.` → `KÙ.BABBAR`

#### C-5. 数値正規化（入力）

- “よくある小数表現” を Unicode 分数へ:
  - `0.8333`→`⅚`, `0.6666`→`⅔`, `0.3333`→`⅓`, `0.1666`→`⅙`
  - `0.625`→`⅝`, `0.75`→`¾`, `0.25`→`¼`, `0.5`→`½`
- **長い小数（小数点以下4桁以上）**は、`lb-35-9` の `_canon_decimal()` で **分数に寄せた丸め**を行う。
  - 小数部分が `1/6, 1/4, 1/3, 1/2, 2/3, 3/4, 5/6` に十分近い場合は、その既定表現（例: `0.33333`）に寄せる
  - それ以外は `%.5f` を基準に末尾の `0` を削る（`rstrip`）
  - ただし **帯分数（例: `1.3333`→`1 1/3` や `1⅓`）への変換はしていない**。
    - 実装上は「整数部 + 小数（`.33333` など）」の形に正規化されるだけで、`1.3333` は概ね `1.33333` の表記に寄る。
    - `⅓` 等への置換は `0.3333` のような **小数単体**でのみ発火し、`1.3333` には直接は適用されない。

#### C-6. 空白正規化

- 連続空白を 1 個にし、前後の空白を削除（`strip`）

## 2) Translation（出力側）— 置換観点

### A. host（@deeppast）推奨（Entry `678899`, 2026-02-26）

- **注釈の除去（訳文側）**
  - `fem.`, `sing.`, `pl.`, `plural`, `(?)` など
  - `<< >>`, `< >`（ただし `<gap>` を除く）や、`..`, `?`, `x`, `xx` 等の stray marks
- **`/` で併記された代替訳の扱い**
  - 例: `"you / she brought"` → `"you brought"`（どちらか片方を選ぶ/落とす）
- **置換**
  - `PN` → `<gap>`
  - `-gold` → `pašallum gold`
  - `-tax` → `šadduātum tax`
  - `-textiles` → `kutānum textiles`
- **数値（分数/小数）正規化**
  - 一部の `x / 12` 系（shekel）を言い換え
  - 小数→Unicode分数（`0.5`→`½` 等）
- **月のローマ数字→整数**
  - 例: `month V` → `month 5`
- **除去しない（test にもある想定）**
  - `"`（直線の引用符）, `'`（アポストロフィ）
  - “意味のある” `?` / `!`

### B. 実装例（`lb-35-9` ノート）に見られる追加の出力整形

`notebooks/006/lb-35-9-ensembling-post-processing-baseline.ipynb` の `VectorizedPostprocessor`（= この repo 内での **実装の一次情報**）は、候補文（複数）をまとめて正規化する。処理の要点は次の通り。

#### B-1. Gap / `PN` / 商品タグの正規化（出力）

- 欠損表現を `<gap>` に統一（入力と同じ `_GAP_UNIFIED_RE`）
- `PN` → `<gap>`
- `-(gold|tax|textiles)` を次へ置換:
  - `-gold` → `pašallum gold`
  - `-tax` → `šadduātum tax`
  - `-textiles` → `kutānum textiles`

#### B-2. shekel 系の定型言い換え（出力）

- `5 11/12 shekels` → `6 shekels less 15 grains`
- `5/12 shekels` → `⅔ shekel 15 grains`
- `7/12 shekels` → `½ shekel 15 grains`
- `1/12 (shekel)` / `1/12 shekel` → `15 grains`
- 派生（自前拡張）: `notebooks/002/[2-3-4]dpc-starter-train-v3.ipynb`
  - `N 11/12 shekels` / `N 11/12 (shekel)` → `N+1 shekels less 15 grains`
  - `N 1/12 shekels` / `N 1/12 (shekel)` → `N shekels 15 grains`
  - `k/12 shekel(s)` / `k/12 (shekel)`（例: `3/12`）→ `45 grains` 等（`k=1..11` の一部を定型表現へ寄せる）
  - `N k/12 ...`（例: `5 7/12 shekels`）→ `5 shekels ½ shekel 15 grains` のように “整数部 + 定型分数表現” へ寄せる

#### B-3. 数値正規化（出力）

- 入力側と同じ “小数→Unicode分数” と `_canon_decimal()`（長い小数の丸め）を適用
  - **帯分数への変換は未対応**（`1.3333`→`1 1/3` にはならず、基本は `1.33333` のような表記に寄る）

#### B-4. 注釈・文法ラベル除去（出力）

- 括弧内の “soft grammar” を除去（例: `(fem. ...)`, `(pl. ...)`, `(?)`, `(!)` を含むパターン）
- `fem/sing/pl/plural` 等の裸ラベルを除去
- `(?)`（不確実）を除去

#### B-5. stray marks / 代替訳の除去（出力）

- `<<...>>` や `<...>` を削除（ただし `<gap>` は保持）
- “数字ではない `/` の代替訳” を削除（`(?<!\\d) / (?!\\d)\\S+`）
  - 例: `you / she brought` のような後半（`/ ...`）が落ちる
  - 例: `11/12` のような **分数の `/` は対象外**（削除されにくい）

#### B-6. 引用符の扱い（出力）

- curly quotes（`“ ” ‘ ’`）のみ削除（直線の `\"` / `'` は保持）

#### B-7. 月表記の正規化（出力）

- `Month XII` → `Month 12`（I〜XII を整数へ）

#### B-8. `<gap>` 保護つきの禁則文字削除（出力）

- `<gap>` を一時的に退避してから、次の文字を削除し、`" <gap> "` として復元する:
  - `()——<>⌈⌋⌊[]+ʾ;`

#### B-9. 重複の縮約・空白整形（出力）

- 同一語の連続（`word word word`）を 1 回に縮約
- 2〜4語程度の短いフレーズの連続を縮約
- 句読点 `..` / `,,` 等の連続を 1 回に縮約
- 句読点の前の余分な空白を削除（例: `"word ."` → `"word."`）
- 連続空白を 1 個にして `strip`

## 3) 運用メモ（比較可能性）

- 置換は最低でも次の 2 系統に分けて ablation できる形に固定するのが安全:
  - `B`: host 推奨（test 整合）中心
  - `C`: ノートブック独自の追加クリーニング込み（攻撃的）
- “入力側の正規化” と “出力側の正規化” は評価（chrF++/BLEU）に与える影響が異なるため、**片側だけON/OFF**できる設計が望ましい。

補足:

- `lb-35-9` では、後処理は **最終出力に対して1回**ではなく、**候補プール（複数候補）に一括適用→MBR選択**の順で使われている（= “候補同士の距離” にも影響する）。
